# PEACEMINUSONE 아카이브 — 팬메이드 콘셉트 사이트 기획서

> 개인 소장/포트폴리오용 비공식 팬페이지. 판매 기능 없음.
> 클로드 코드와 Phase 단위로 진행하기 위한 문서.

---

## 1. 한 줄 소개

브랜드가 지나온 길(전시 → 설립 → 협업)과 제품을 기록하는 다크 톤의 아카이브 사이트. 쇼핑몰이 아니라 "기록관".

## 2. 콘셉트 원칙

- **다크 베이스** (#0c0c0c 근처), 타이포그래피 중심, 여백으로 말하기.
- **첫 화면은 타이포만.** 대표 이미지는 스크롤 한 번 아래에서 풀스크린 "챕터"로 등장.
- **"minus one" 모티프를 레이아웃 언어로**: 아카이브 그리드에서 한 칸을 의도적으로 비움.
- **Shop 페이지는 존재하되 평소엔 'closed' 상태.** 텅 빈 화면 + "closed — drops are unannounced" 한 줄. 실제 브랜드의 기습 드롭 문법을 UI로 번역한 것.
- 푸터에 "unofficial fan archive — not affiliated" 명시.
- 애니메이션은 절제: 느리고 확신 있는 페이드/라이즈, 스크롤 기반 등장.

## 3. 페이지 구조

| 페이지 | 내용 |
|--------|------|
| Home | 타이포 히어로 → 풀스크린 이미지 챕터(2015 전시) → 셀렉티드 아카이브 그리드 |
| Archive | 전 제품 그리드 (연도/카테고리 필터), 한 칸 비움, 클릭 시 상세 |
| Timeline | 2015 전시 'Beyond the Stage' → 2016.10 설립 → 주요 협업 → 현재. 스크롤 따라 연도별 챕터 등장 |
| Philosophy | 브랜드 의미(이상과 현실의 교차점)를 시처럼 한 페이지로 |
| Shop | 기본 'closed' 상태 화면. (선택) 공식몰 외부 링크 한 줄 |

## 4. 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Astro (또는 순수 HTML/CSS/JS) | 정적 사이트, 빠르고 가벼움 |
| 애니메이션 | GSAP + ScrollTrigger | 스크롤 기반 챕터 등장, 패럴랙스 |
| 데이터 | `data/products.json`, `data/timeline.json` | 콘텐츠와 코드 분리 |
| 폰트 | 산세리프 1종 + 포인트 세리프(이탤릭) | 타이포 중심이므로 신중히 |
| 배포 | 안 함(로컬) 또는 비공개 링크 | 개인 소장용 |

## 5. 에셋 파이프라인 (핵심)

사이트가 특정 이미지에 의존하지 않게 만든다. 이미지는 폴더에 넣기만 하면 자동 반영.

```
assets/
  brand/
    logo.png          ← 직접 저장한 로고 파일을 여기에 (없으면 텍스트 로고로 폴백)
  archive/
    2016_cap_first-drop.jpg
    2017_tee_daily.jpg
    ...               ← 파일명 규칙: {연도}_{카테고리}_{이름}.jpg
  chapters/
    2015_exhibition.jpg   ← 풀스크린 챕터용 가로 이미지
```

- `data/products.json` 스키마:

```json
{
  "id": "2016_cap_first-drop",
  "year": 2016,
  "category": "cap",
  "title_ko": "퍼스트 드롭 캡",
  "caption": "브랜드의 시작을 알린 첫 발매",
  "image": "assets/archive/2016_cap_first-drop.jpg"
}
```

- 클로드 코드에 **폴더 스캔 → products.json 초안 자동 생성 스크립트**를 만들게 할 것. (파일명 규칙에서 연도·카테고리 파싱)
- 이미지가 없는 항목은 다크 플레이스홀더로 표시 → 사이트는 항상 완성 상태 유지.

> 에셋 수집은 본인 담당: 공식 사이트/공식 인스타그램에서 직접 저장, 소장품 직접 촬영. 과거 사이트의 연혁 구성은 웨이백머신(web.archive.org)에서 peaceminusone.com 검색해 참고.

## 6. 단계별 작업 순서 (클로드 코드에 이 순서로 요청)

### Phase 0 — 셋업
- Astro 프로젝트 생성(또는 정적 HTML 구조), GSAP 설치
- 다크 베이스 + 타이포 스케일 + 공통 레이아웃(헤더/푸터)
- 헤더: `assets/brand/logo.png` 있으면 이미지, 없으면 레터스페이싱 넓은 텍스트 로고

### Phase 1 — Home 히어로 & 챕터
- 타이포 히어로(순차 라이즈 인 애니메이션) + "scroll ↓"
- 풀스크린 이미지 챕터: 스크롤 진입 시 서서히 밝아지며 살짝 확대(패럴랙스)
- ✅ 검증: 첫 화면 타이포 → 스크롤 → 이미지 한 방의 리듬이 사는지

### Phase 2 — 에셋 파이프라인
- 폴더 스캔 → products.json 자동 생성 스크립트
- 플레이스홀더 폴백 처리
- ✅ 검증: 이미지 2~3장만 넣고 그리드가 자동으로 채워지는지

### Phase 3 — Archive 페이지
- 연도/카테고리 필터, 그리드(한 칸 비움 고정), 호버 시 캡션
- 클릭 → 상세(큰 이미지 1장 + 연도 + 짧은 캡션)

### Phase 4 — Timeline 페이지
- timeline.json 기반, 스크롤 따라 연도 챕터가 하나씩 등장 (ScrollTrigger)
- 텍스트 최소화: 연도 + 한 줄

### Phase 5 — Philosophy & Shop(closed)
- Philosophy: 한 페이지 시 형식, 세리프 이탤릭 포인트
- Shop: 기본 'closed' 화면. 비어 있음 자체가 디자인

### Phase 6 — 마감
- 반응형(모바일), 폰트/간격 미세조정, 페이지 전환 페이드
- 포트폴리오용 화면 녹화/캡처

## 7. 포트폴리오 설명 포인트

- "브랜드의 철학(minus one)을 레이아웃 언어로 번역" — 그리드의 빈 칸
- "브랜드의 판매 방식(기습 드롭)을 UI로 번역" — 평소 closed인 Shop
- "이미지에 의존하지 않는 에셋 파이프라인" — 콘텐츠가 늘어도 구조 불변
- 비공식 팬 아카이브임을 명시하고 개인 소장/설명용으로만 사용

## 8. 첫 프롬프트 예시 (클로드 코드)

```
피스마이너스원 비공식 팬 아카이브 사이트를 만들 거야. 첨부한 기획서의 Phase 0부터.
Astro 프로젝트를 만들고 다크 베이스(#0c0c0c)와 타이포 스케일, 공통 헤더/푸터를 잡아줘.
헤더 로고는 assets/brand/logo.png가 있으면 이미지, 없으면 텍스트 로고로 폴백되게.
```
